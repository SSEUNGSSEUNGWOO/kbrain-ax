# KBrain-AX — AI·데이터 전문인재 선발 통합 플랫폼

> 서면평가부터 온라인 시험, 결과 관리까지 하나의 플랫폼에서

---

## 1. 개요

KBrain-AX는 AI·데이터 전문인재 선발 과정 전반을 통합 관리하는 웹 플랫폼입니다.
기존에 분리 운영되던 서면평가 시스템과 온라인 시험 시스템을 단일 플랫폼으로 통합하여 운영 효율성과 안정성을 높이는 것을 목적으로 합니다.

---

## 2. 기존 운영 문제점

### 2-1. 시스템 분리 운영
| 구분 | 기존 시스템 | 문제점 |
|------|------------|--------|
| 서면평가 | evalu-lens (별도 사이트) | 평가 결과 수동 취합 필요 |
| 온라인 시험 | aicapa.kr (별도 사이트) | 시험 결과 별도 관리 |
| 통합 관리 | 없음 | 두 시스템 결과를 엑셀 등으로 수동 병합 |

### 2-2. 온라인 시험 운영 장애 (실 운영 사례)
200명 이상 동시 응시 환경에서 다음과 같은 오류가 발생했습니다:

#### 시스템 불안정
- 일부 수험자 접속 튕김 현상 발생
- 텍스트 입력이 되지 않는 현상 발생
- **원인**: 답안 자동저장이 짧은 주기로 서버에 반복 요청되어 DB 연결 포화

#### 타이머 오작동
- 시간이 남아있음에도 시험이 강제 종료되는 현상
- **원인**: 타이머를 클라이언트(브라우저) 기준으로 관리하여, 탭 전환·백그라운드 전환 시 브라우저의 타이머 스로틀링이 적용되어 오작동 발생

#### 실시간 감시 시스템 불안정
- 200명 웹캠 스트림을 실시간으로 수집·표출하는 구조로 운영
- 웹캠·마이크·화면공유·탭전환 감지 기능 불안정
- **원인**: 전용 미디어 서버 없이 브라우저에서 직접 WebRTC 연결을 처리하여 200개 이상 동시 스트림 시 클라이언트 및 서버 과부하 발생

## 3. KBrain-AX 도입 목적 및 개선 방향

| 기존 문제 | 개선 방향 |
|----------|----------|
| 서면평가·시험 시스템 분리 운영 | 단일 플랫폼 통합, 결과 자동 취합 |
| 주관적 평가 기준 | 루브릭 기반 항목별 채점으로 일관성 확보 |
| DB 연결 포화로 인한 입력 오류 | 답안 로컬 임시저장 + 30초 주기 서버 동기화 |
| 클라이언트 타이머 오작동 | 서버 시작 시각 기준 경과시간 계산 방식으로 변경 |
| 실시간 웹캠 감시 시스템 불안정 | 단기: Zoom 병행 운영 / 장기: Agora WebRTC 서비스 연동 검토 |
| 대규모 동시접속 불안정 | Supabase Connection Pooling + Vercel 클라우드 인프라 활용 |

## 4. 시스템 구성

| 구성요소 | 기술스택 | 역할 |
|----------|---------|------|
| 프론트엔드 | Next.js 14, Tailwind CSS, shadcn/ui | UI 전체 |
| 백엔드/DB | Supabase (PostgreSQL) | 데이터 저장 및 API |
| 인증 | Supabase Auth + Google OAuth | 로그인 |
| 배포 | Vercel | 호스팅 |

### DB 테이블 구성
- `profiles` — 사용자 정보 및 역할(admin / applicant)
- `selections` — 선발 전형 정보
- `applications` — 지원서
- `evaluations` — 서면평가 결과
- `exams` — 시험 정보
- `exam_questions` — 시험 문항
- `question_bank` — 문제 은행
- `exam_attempts` — 시험 응시 기록

## 6. 동시 접속 수용 가능 인원 (보수적 기준)

| 구간 | 안정성 | 비고 |
|------|--------|------|
| 150명 이하 | 안정 | 무료 플랜 충분 |
| 150~300명 | 대체로 안정 | 파일럿 테스트 필수, 무료 플랜 가능 |
| 300명 이상 | 불안정 가능 | Supabase Pro 권고 |

> Supabase JS 클라이언트는 REST API 방식으로 DB 직접 연결 수 제한과 무관하나, 시험 시작/종료 시 동시 요청 폭발 구간에서 응답 지연 발생 가능
> 자동저장 주기를 30초 이상으로 설계하면 요청 분산 효과로 수용 인원 확대 가능

## 7. 예상 운영 비용 (300명 동시 시험 기준)

### A. 상시 운영비 (월 고정)
| 항목 | 서비스 | 비용 |
|------|--------|------|
| 프론트엔드 호스팅 | Vercel Pro | $20/월 |
| DB / 인증 | Supabase Pro | $25/월 |
| **월 합계** | | **$45/월 (약 6만원)** |

> 플랜 변경 없이 안정적 운영 가능, 평시 데이터 관리 및 테스트 환경 유지

### B. 시험·평가 운영비 (전형 1회당 발생)
| 항목 | 서비스 | 비용 |
|------|--------|------|
| 실시간 웹캠 감시 | Agora (300명 × 1시간) | 약 $12 (약 1만7천원) |
| 지원서 AI 평가 | Claude API (300명, 루브릭 7항목 × 3회 일관성 검증) | 약 $150 (약 21만원) |
| **전형 1회 합계** | | **약 $162 (약 23만원)** |

> Agora 월 10,000분 무료 제공 → 초과분 $0.00099/분
> 300명 × 60분 = 18,000분 → 초과 8,000분 × $0.00099 ≈ $8
>
> Claude API (claude-sonnet-4): 입력 $3/M, 출력 $15/M
> 1인당 평가: 루브릭+지원서 입력 ~30,000토큰 + 결과 출력 ~10,500토큰 × 3회 일관성 검증 + 피드백 생성
> → 1인당 ~$0.50 (여유 기준), 300명 × $0.50 = **$150**
> 전형이 없는 달은 비용 미발생

### C. 초기 구축비 (최초 1회)
| 항목 | 비용 |
|------|------|
| 도메인 (.kr / .com) | 연 1~2만원 |


### 웹캠 감시 서비스 선택 근거
- **Agora 채택**: 전 세계 200개국 서비스, 99.9% SLA 보장, 카카오·라인 등 국내 대형 서비스 사용 중
- 기존 시스템(aicapa.kr) 장애 원인은 자체 WebRTC 구현의 한계였으며, Agora는 수만 명 동시 처리 인프라 보유
- 300명은 Agora 기준 소규모 트래픽으로 안정적 운영 가능

## 9. 개발 계획 및 일정

현재 AI·데이터 전문인재 선발 기준으로 자체 MVP 모델을 개발 완료하였으며, 5월 첫째 주 내 핵심 기능 개발을 마무리하고 5월 중순부터 실제 운영 환경에서 테스트를 진행할 예정입니다.
